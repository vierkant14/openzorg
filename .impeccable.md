# OpenZorg Design System

## Design Context

### Users
OpenZorg serves multiple primary user types across Dutch healthcare organizations:
- **Wijkverpleegkundigen / Verzorgenden** — On tablets during home visits, need quick access to client info, fast rapportage entry. Often standing, gloves on, limited attention. Nachtdiensten also.
- **Planners** — Desktop, large screens. Need clear calendar/timeline views, drag-and-drop, dense information without clutter.
- **Functioneel beheerders** — Desktop. Configure the system: custom fields, validation rules, workflows. Need a Power Apps-like experience where configuration feels empowering, not technical.
- **Teamleiders** — Quick overview dashboards, caseload, quality indicators.

Context: used 24/7 including night shifts. Must support both light and dark mode based on time/preference.

### Brand Personality
**Betrokken. Voor de mens. Met elkaar.**

Three brand words: *warm, menselijk, betrokken* (warm, human, involved)

OpenZorg should feel like a colleague who cares — not a system that demands. The interface should disappear into the workflow, not impose itself.

Emotional goals:
- Trust and safety (healthcare data)
- Warmth and approachability (not cold/clinical)
- Clarity and calm (not overwhelming)
- Empowerment (configuration should feel creative, not restrictive)

### Anti-references
- SAP — overwhelming, system-driven, hostile to humans
- HiX — dense, clinical, feels like it was designed by engineers for engineers
- Any ECD that forces users through rigid flows instead of supporting their natural workflow
- Generic "admin dashboard" templates with metric cards and sparklines

### References
- **Knab** (banking app) — for care users. Simple, warm, task-focused. You do what you need and leave.
- **TickTick** — for planners. Clean agenda/task views, visual but dense, good use of color for status.
- **Power Apps + Power Automate** — for beheerders. Visual configuration, drag-and-drop, feels like building not coding.

### Aesthetic Direction

**Tone**: Warm professional. Not sterile-clinical, not playful-startup. Think: a well-designed care facility — calming colors, natural materials, clear signage, human warmth.

**Color direction** (inspired by Intilifi logo):
- Primary: Teal/turquoise — calming, associated with healthcare, trust
- Secondary: Deep navy — authority, stability
- Accents: Warm coral/salmon — humanity, warmth, attention
- Neutrals: Warm-tinted grays (not cold blue-grays)

**Theme**: Auto light/dark based on system preference. Light mode primary (daytime care), dark mode essential (nachtdienst).

**Typography direction**: Readable, warm, slightly rounded. Not cold geometric sans-serifs. Should feel approachable on tablet at arm's length.

### Design Principles

1. **Mensen eerst** — Every screen asks: "does this help the zorgverlener care for their client?" If not, remove it.
2. **Rust in overzicht** — Calm clarity. Dense information presented with breathing room. No visual noise.
3. **Configureer, niet programmeer** — Admin experiences should feel visual and empowering, like building with blocks.
4. **Eén klik minder** — Reduce clicks relentlessly. Progressive disclosure, not deep navigation trees.
5. **24/7 leesbaar** — Works in bright daylight on a tablet and at 3AM on a desktop in a dimly lit office.
